## 프로젝트 개요 : 폴더를 타이머로 감시하고 있다가 텍스트 파일이 생성되면 이미지를 한장씩 인코딩해서 전송하고 응답받는 것을 반복한다음 모든 이미지가 처리되면 텍스트 파일을 삭제하는 프로그램

프로그램이 시작되면 실행파일 옆에 있는 OCRChecker.ini 파일을 읽는다.

[CHECK_FOLDER]
PATH=C:\ocr

[IMAGE_FOLDER]
PATH=C:\img

[OCR_SERVER]
URL=localhost:18333

해당 값을 변수로 저장한다.

GUI 화면에는 버튼 2개가 있다. 강제시작 버튼과 강제취소 버튼.

타이머를 시작시켜 5초에 한번씩 CHECK_FOLDER 의 파일 목록을 감시한다.
파일 중 ocr.txt 파일이 발견되면 타이머를 멈추고 IMAGE_FOLDER 경로의 이미지 파일(png, jpg) 을 1개 읽어서 base64로 인코딩한다(만약 동일 파일명이며 뒤에 ^0001 다음의 숫자로 있으면 연달아서 인코딩한다).
그 후 해당 파일(파일들)은 삭제한다.

<<예시>>
[
	{"filename": "우리집^0001.png", "data": "base64string..."}
]

[
	{"filename": "이웃집^0001.png", "data": "base64string..."}
]

[
	{"filename": "친구집^0001.png", "data": "base64string..."},
	{"filename": "친구집^0002.png", "data": "base64string..." },
	{"filename": "친구집^0003.png", "data": "base64string..." }
]

인코딩한 파일명과 base64 문자열을 json 형태로 OCR_SERVER에 POST요청으로 전송하고 응답을 기다린다. 타임아웃 10분
(만약 동일한 파일명에 연관된 숫자파일이 있으면 같이 배열로 묶어서 전송한다. data가 길 수 있으니 짤리더라도 계속해서 연달아 보낼 수 있도록 처리한다)


서버에서 처리 후 응답이 오면 응답 문자열 JSON 배열을 CHECK_FOLDER의 활동데이터_견적서템플릿.xlsx 파일에 기록한다. (각각 한줄씩)

더이상 이미지 파일이 없으면 ocr.txt 파일을 삭제한다.

끝나면 다시 타이머를 시작한다.

*만약 처리 중 강제취소 버튼을 누르면 서버에 취소 메시지를 전송하고 ocr.txt 파일을 삭제한다. 그 후 다시 타이머를 시작한다.*
*현재 처리중인지 감시중인지 알 수 있도록 텍스트로 출력한다.*
